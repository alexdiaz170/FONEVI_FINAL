### 2. Colas de Tareas (BullMQ + Redis)

**Impacto:** Availability 650↑, Fault Tolerance 580↑, Performance 700↑

| Archivo                                      | Propósito                                                                |
| -------------------------------------------- | ------------------------------------------------------------------------ |
| `src/infrastructure/queue/JobQueue.ts`       | Queue factory + Worker manager con fallback inline si no hay Redis       |
| `src/infrastructure/queue/whatsappWorker.ts` | Worker de WhatsApp — envía mensajes en background                        |
| `src/infrastructure/queue/reportWorker.ts`   | Worker de reportes — genera PDF/Excel fuera del request                  |
| `src/index.ts`                               | Inicializa workers al arrancar + `shutdownQueues()` en graceful shutdown |

**WhatsApp:** `POST /api/whatsapp/enviar` encola el mensaje y responde inmediatamente con `{ encolado: true, jobId }`. Worker procesa en background con 3 reintentos exponenciales. Sin Redis → ejecución inline (comportamiento anterior).

**Reportes:** `GET /api/exportar/:tipo/:formato?async=true` encola la generación. Sin `async` → síncrono (comportamiento anterior). `GET /api/exportar/descargar/:filename` sirve archivos generados.
